# Product

## Register

product

## Users

正在学习使用 KWDB 的开发者。他们希望通过智能电表示例了解 KWDB 如何承载关系数据、时序数据和跨模型查询，并在探索示例数据库、查看数据结构、运行示例 SQL 的过程中建立对 KWDB 应用方式的理解。

## Product Purpose

这个产品通过真实感较强的智能电表场景演示 KWDB。它应帮助开发者快速看清系统中有哪些数据、数据之间如何关联、示例 SQL 如何体现 KWDB 在智能电表分析中的价值。成功状态是：开发者无需和界面较劲，就能理解数据模型和查询模式。

## Brand Personality

专业、清晰、收敛。界面应像一个聚焦的技术学习工具：可信、冷静，并且明确服务于 KWDB 示例，而不是通用后台模板。

## Anti-references

避免看起来像模板生成的后台管理系统。避免通用仪表盘装饰、嘈杂的小组件网格、装饰性渐变，以及让示例显得像占位产品而不是 KWDB 学习体验的 UI 模式。

## Design Principles

1. 用真实界面完成教学：页面应通过真实数据和查询呈现 KWDB 概念，而不是依赖脱离操作的说明文字。
2. 保持路径清晰：开发者应始终知道自己在哪里、正在看什么数据、下一步可以尝试什么操作。
3. 技术具体性优先于通用后台感：标签、空状态和视觉层级都应强化智能电表与 KWDB 示例语境。
4. 在信息密度中保持收敛：可以使用熟悉的产品 UI 模式，但要调校到像经过设计的学习工具，而不是套壳模板。
5. 让状态和失败可处理：连接、加载、离线、查询错误等状态应说明发生了什么，以及开发者可以如何恢复。

## Accessibility & Inclusion

目标符合 WCAG AA。正文、控件、图表和状态指示应满足对比度要求，支持键盘访问，保留清晰的焦点状态，并避免只依赖颜色传达信息。动效应尊重减少动态效果偏好。
